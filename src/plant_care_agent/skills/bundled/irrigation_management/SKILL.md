# 灌溉管理 SOP

## 决策流程

### 1. 数据采集
- 调用 `read_sensors` 获取目标地块的土壤湿度、空气温度、降雨量
- 调用 `read_sensor_history` 查看过去 24-48 小时的湿度变化趋势
- 调用 `weather_forecast` 查看未来 7 天降雨预报

### 2. 灌溉判断标准

| 土壤湿度 | 动作 | 说明 |
|----------|------|------|
| < 20% | 🔴 紧急灌溉 | 作物面临干旱胁迫 |
| 20-30% | 🟡 建议灌溉 | 即将进入缺水区间 |
| 30-60% | 🟢 适宜 | 无需灌溉 |
| 60-80% | 🟡 偏湿 | 暂停灌溉，加强排水 |
| > 80% | 🔴 过湿 | 检查排水系统 |

### 3. 灌溉量计算参考
- 沙土：每亩 20-30 m³
- 壤土：每亩 30-40 m³
- 黏土：每亩 40-50 m³
- 高温天气（>35°C）增加 20%
- 未来 2 天有降雨（概率 >60%）减少 30-50%

### 4. 执行
- 使用 `propose_operation` 提出灌溉建议，包含水量和时长
- 等待用户确认
- 确认后 `execute_operation` 执行
- 使用 `log_farm_event` 记录灌溉事件

### 5. 效果验证
- 灌溉后 1-2 小时再次 `read_sensors` 检查土壤湿度变化
- 生成 `sensor_trend` 图表查看灌溉前后对比
