# 植物巡检 Skill

按以下步骤对目标植物执行全面巡检。每一步调用对应工具获取**原始数据**，由你（LLM）做专业分析和判断。

## 0. 确定巡检范围

- 用户指定了某棵植物 → 只巡检那棵
- 用户说"巡检一下"、"看看所有植物" → 先 `plant_project__list_plant_projects` 获取列表，逐棵巡检
- 如果没有任何植物项目，提示用户先用 `plant_project__create_plant_project` 创建

## 1. 天气风险评估

调用 `plant_inspect_tools__get_weather_assessment`，输入植物名称。

拿到数据后分析：
- 天气代码 96/99 → **冰雹**，立即建议移入室内
- 风速 > 75km/h → **大风**，固定花盆、脆弱植物入室
- 风速 > 50km/h → **强风**，注意固定
- 最低温 < 0°C → **冻害**，必须入室或覆盖保温
- 最低温 < 5°C → **低温**，建议保温
- 最高温 > 40°C → **极端高温**，必须遮阴充分浇水
- 最高温 > 35°C → **高温**，注意遮阴
- 降水 > 50mm → **强降水**，户外植物避雨
- 降水 > 30mm → **暴雨**，盆栽考虑移位
- 有降雨 → 可减少浇水

## 2. 养护状态检查

调用 `plant_inspect_tools__get_care_status`，输入植物名称。

拿到数据后分析：
- **浇水**：对比「距上次浇水天数」vs「设定浇水间隔」，超期则提醒
- **施肥**：
  - 检查当前月份是否在「免施肥月份」内 → 是则跳过
  - 对比「距上次施肥天数」vs「设定施肥间隔」，超期则提醒
  - 提醒时附上「推荐肥料类型」
  - 如果养护参数未设置（显示"未设置"），建议调用 `plant_project__set_care_schedule` 配置
- **周末**：周六建议修剪枯叶、检查整体状态

## 3. 生长进展评估

调用 `plant_inspect_tools__get_growth_status`，输入植物名称。

拿到数据后分析：
- 种植超过一周但记录很少 → 建议定期记录
- 播种期超过 14 天 → 确认是否已发芽
- 根据生长阶段给出阶段性建议（苗期、生长期、花期、果期、休眠期各有不同）
- 查看最近事件，判断养护是否规律

## 4. 病虫害风险评估

调用 `plant_inspect_tools__get_pest_risk_factors`，输入植物名称。

拿到数据后分析：
- 高温(>25°C) + 高湿(>75%) → 真菌/虫害高风险
- 温湿偏高(>20°C / >65%) → 中风险
- 苗期或花期 → 易感阶段
- 近期有病虫害记录(14天内) → 复发风险
- 连续降雨(3天中≥2天>5mm) → 真菌风险

风险等级：
- 高风险（≥3 个因素）→ 检查叶片、保持通风、考虑预防性用药
- 中风险（≥2 个因素）→ 检查叶片、改善通风
- 低风险 → 正常养护

## 5. 综合报告

将以上 4 项汇总为结构化报告：

```
🌱 「{植物名}」巡检报告

🌤 天气: {正常/需关注/紧急} — {简要描述}
🪴 养护: {正常/需关注} — {简要描述}
📈 生长: {简要描述}
🐛 病虫害: {低/中/高} — {简要描述}

📋 待办事项:
1. {最紧急的行动}
2. {次要行动}
...
```

## 6. 记录巡检（可选）

如果发现异常，建议用 `growth_journal__log_event` 记录巡检结果：
```
{植物名} | 巡检 | {巡检发现摘要}
```

## 多植物汇总

如果巡检了多棵植物，最后做一个全局汇总：
- 按紧急程度排序
- 标注共同问题（如天气影响所有植物）
- 给出优先处理建议
